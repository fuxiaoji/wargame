# 《追击俾斯麦》DRL 时空状态张量设计规范 (Tensor Design Specification)

## 1. 架构总览 (Architecture Overview)

本系统采用**集中式评判、分布式执行 (Centralized Critic, Decentralized Actors, CTDE)** 的多智能体强化学习架构。环境引擎在每一个时间步，首先生成一个包含绝对真实信息的“上帝视角时空张量 \$S\_{global}\$”，随后通过不对称掩码（Masks）生成英军视角张量 \$O\_{brit}\$ 和德军视角张量 \$O\_{ger}\$。

张量的核心维度为标准 5D 时空序列张量：

**`Input Shape = [Batch_Size, Time_Steps, Channels, Height, Width]`**

具体至本环境，其尺寸严格定义为：**`[B, 73, 128, 8, 6]`**

### 1.1 维度物理解析

* **`Height = 8`, `Width = 6`**: 对应《追击俾斯麦》物理棋盘（A-F共6列，1-8共8行）。
* **`Channels = 128`**: 包含地形、实体状态、历史线索与认知预测的特征维度。
* **`Time_Steps = 73`**: 基于规则严格拆分的马尔可夫状态转移步长。
  * 游戏共 18 回合，每回合严格分为 **4 个决策阶段**（1. 德军移动; 2. 英军移动; 3. 航空索敌; 4. 战斗/攻击航路）。
  * 总时间步 = \$18 \\times 4 + 1(\\text{初始部署阶段}) = 73\$ 步。

## 2. 128 维通道 (Channels) 详细分配映射表

128 个 Channel 划分为 5 个逻辑区块 (Blocks)。以下定义均基于上帝视角 \$S\_{global}\$。

### Block 1: 静态地理与全局态势 (Channels 0 - 15)

*此区块对双方完全公开，无掩码。*

| **Channel** | **特征名称 (Feature)** | **物理意义与数值映射**                       |
| ----------------- | ---------------------------- | -------------------------------------------------- |
| **0**       | `Navigable_Mask`           | 陆地/无编号区 = 0.0，可通行海域 = 1.0              |
| **1**       | `Convoy_Routes`            | 规则 8.0：运输航路格 = 1.0，其余 = 0.0             |
| **2**       | `Brest_Target`             | 德军胜利目标港口：F7 = 1.0，其余 = 0.0             |
| **3**       | `German_Spawns`            | 德军初始可能出生点：A5, A6, B7 = 1.0               |
| **4**       | `Phase_GerMove`            | 当前阶段是否为“德军移动阶段”：全图填 1.0 或 0.0  |
| **5**       | `Phase_BritMove`           | 当前阶段是否为“英军移动阶段”                     |
| **6**       | `Phase_BritSearch`         | 当前阶段是否为“英军索敌阶段”                     |
| **7**       | `Phase_Combat`             | 当前阶段是否为“战斗/破袭阶段”                    |
| **8**       | `Turn_Progress`            | 游戏进度标量：当前回合数 / 18.0 (铺满全图)         |
| **9**       | `VP_British`               | 英军当前胜利点数 / 6.0 (归一化，铺满全图)          |
| **10**      | `VP_German`                | 德军当前胜利点数 / 6.0 (归一化，铺满全图)          |
| **11-15**   | `Reserved_Global`          | 预留给寻路距离图 (Distance Field) 等静态启发式特征 |

### Block 2: 英军实体中心化特征 (Channels 16 - 47)

*每类高价值单位分配一组专属 4 维特征 `[位置(0/1), 当前HP, 攻击力, 机动锁定状态(0/1)]`。*

| **Channel** | **特征名称 (Feature)** | **物理意义与数值映射**                                              |
| ----------------- | ---------------------------- | ------------------------------------------------------------------------- |
| **16-19**   | `Hood_State`               | **胡德号**- C16:位置, C17:HP(1-2), C18:攻击力, C19:锁定(1=禁步)     |
| **20-23**   | `PoW_State`                | **威尔士亲王号**- 同上结构                                          |
| **24-27**   | `ArkRoyal_State`           | **皇家方舟号**- 同上结构 (其机动锁定 C27 恒为 0)                    |
| **28-31**   | `Generic_2Step`            | **其他2步战舰**- 该格内舰船数量聚合、均血量、均攻击力、均锁定状态   |
| **32-35**   | `Generic_1Step`            | **其他1步战舰**- 同上结构                                           |
| **36-39**   | `Dummy_State`              | **伪装算子**- C36:数量, C37:HP(恒为1), C38:攻(0), C39:移动力(恒为3) |
| **40**      | `Brit_Anonymous_Pos`       | **【专供德军通道】**叠加所有英军背面朝上算子的位置，全图铺满 1/0          |
| **41-47**   | `Reserved_Brit`            | 英军扩展预留通道                                                          |

### Block 3: 德军实体中心化特征 (Channels 48 - 63)

*同样采用专属 4 维特征 `[位置, 当前HP, 攻击力, 移动力]`。*

| **Channel** | **特征名称 (Feature)** | **物理意义与数值映射**                                                                   |
| ----------------- | ---------------------------- | ---------------------------------------------------------------------------------------------- |
| **48-51**   | `Bismarck_State`           | **俾斯麦号**- C48:位置(唯一标1的格), C49:剩余Step(1-4), C50:攻, C51:移动力(满血2, 破损1) |
| **52-55**   | `PrinzEugen_State`         | **欧根亲王号**- (扩展变体预留) 同上结构                                                  |
| **56-63**   | `Reserved_Ger`             | 德军扩展预留通道                                                                               |

### Block 4: 战场观测、遗迹与战争迷雾线索 (Channels 64 - 95)

*记录“信息泄露事件”与马尔可夫动量特征（衰减轨迹）。*

| **Channel** | **特征名称 (Feature)** | **物理意义与数值映射**                                |
| ----------------- | ---------------------------- | ----------------------------------------------------------- |
| **64**      | `Event_CombatReveal`       | 本回合因同格索敌导致俾斯麦暴露的绝对坐标 (标1)              |
| **65**      | `Event_RadarReveal`        | 本回合航母航空侦察扫出的俾斯麦坐标 (标1)                    |
| **66**      | `Event_ConvoyReveal`       | 德军攻击运输船导致信号泄露的坐标 (标1)                      |
| **67**      | `Event_DummyKill`          | 德军投骰子鉴定并摧毁伪装算子的坐标 (极重要追踪线索)         |
| **68**      | `Fog_Cleared`              | 英军本回合排查过且宣告“未发现”的格子 (绝对安全区)         |
| **69-79**   | `Ger_Pheromone`            | 德军历史轨迹衰减（动量）。如：上一步格填 1.0，上两步 0.8... |
| **80-95**   | `Reserved_Obs`             | 预留给天气视距限制等观测变化                                |

### Block 5: 认知网络草稿区 (Channels 96 - 127)

*用于存放前向传播中，底层模型推导出的高阶特征，供后置 EUA 与 Transformer 读取。*

| **Channel** | **特征名称 (Feature)** | **物理意义与数值映射**                                        |
| ----------------- | ---------------------------- | ------------------------------------------------------------------- |
| **96**      | `Belief_State_B`           | 前置预测头输出的德军位置概率热力图矩阵\$\\mathbf{B}\$(仅英军侧有效) |
| **97**      | `Intent_State_Pi`          | 预测出的敌方行动概率矩阵\$\\mathbf{\\Pi}\_{opp}\$![]()<br/>           |
| **98-127**  | `Hidden_Context`           | Transformer 自注意力机制的中间隐状态映射预留                        |

## 3. 双方不对称掩码执行逻辑 (Asymmetric Masking Logic)

物理环境引擎 (Environment) 的 `step()` 函数返回时，必须通过严格的掩码隔离操作，切断数据泄露。

### 3.1 生成英军观测空间 \$O\_\$![]()

**核心逻辑：剥夺德军绝对位置，强迫网络依赖雷达线索 (Block 4) 与预测矩阵 (Block 5)。**

```
O_brit = S_global.copy()
# 1. 彻底掩盖俾斯麦的绝对实体属性 (Block 3 全部清零)
O_brit[:, :, 48:64] = 0.0 
# 2. 掩盖德军走过的历史动量痕迹 (除非该痕迹因事件暴露)
O_brit[:, :, 69:80] = 0.0 

```

### 3.2 生成德军观测空间 \$O\_\$![]()

**核心逻辑：剥夺英军算子的具体身份（语义混淆），保留位置压迫感。**

```
O_ger = S_global.copy()
# 1. 彻底掩盖英军的具体舰船身份和HP数据 (Block 2 中 16-39 全部清零)
O_ger[:, :, 16:40] = 0.0
# 2. 仅向德军开放匿名算子位置 (Channel 40)
# (Channel 40 在引擎生成 S_global 时已预先计算好)
# 3. 掩盖英军大脑里的信念预测矩阵
O_ger[:, :, 96:98] = 0.0

```

## 4. 时空联合建模前向传播 (Forward Pass) 规范

本张量设计支持 **Late-Fusion** 架构：

1. **切片输入**：当前时刻 \$t\$ (如第 25 步)，提取 `[Batch, 1, 128, 8, 6]` 的单步切片。
2. **CNN 空间压缩**：CNN 压平这 128 个 Channel 的空间信息，输出空间特征向量 \$v\_t\$。
3. **记忆拼接**：将历史步长的向量拼接为 `[Batch, t, 256]`，加入 Phase Embedding（通过 C4-C7 识别）。
4. **Transformer 统筹**：进行时序序列自注意力计算，输出 Context Vector。
5. **EUA 层介入**：生成 Channel 96 的 \$\\mathbf{B}\$ 矩阵，代入 EUA 公式计算期望收益，最终输出动作（移动坐标或宣告索敌）。
